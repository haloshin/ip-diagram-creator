# 使用场景、尺寸、模式和图内信息量

## 原则

用户输入内容后，Skill 先主动推荐最适合的图类型、尺寸和图内信息量，但不锁死。用户可以改。

先判断视觉模式，再判断图类型和尺寸。不要把“手绘插图”和“知识卡”混成同一条密度轴。

## 两大视觉模式

### 手绘插图模式

重点是：

- 一个核心认知锚点。
- 一个主要动作隐喻。
- 大量留白。
- 少量中文手写标注。
- 角色参与核心动作。

即便选择较高图内信息量，也只是增加标签、节点、反馈线或风险提示，不要铺满成知识卡。

如果人物只占画面 10%-20%，默认需要参考动作 / 表情 / 小比例场景扩展图。

### 知识卡模式

知识卡是完整内容容器，不是“高密度插图”。它可以承载标题、模块、步骤、风险、行动、金句、评分、截图批注、课程结构等。

知识卡的参考图只提供方向，不是固定模板。根据内容的信息结构、使用场景、尺寸比例和阅读路径，主动匹配或创造更合适的展现形式。

角色分工：

- 用户 IP 通常是主讲人、老师、拆解者、引导者或批注者。
- 辅助角色适合穿插演绎：用户、执行 Agent、失败案例、产品搬运者、反馈来源、风险提示、流程中的小角色。

## 图类型池

| 图类型 | 默认尺寸 | 默认图内信息量 | 出图效果 |
|---|---:|---|---|
| 长文配图 | 16:9 | 画面更空 | 手绘插图模式，只讲一个关键点，文字很少 |
| 图文号知识卡 | 3:4 或 4:5 | 讲清楚重点 | 竖版知识卡，适合图文平台滑动阅读 |
| 手机海报 | 9:16 | 内容更完整 | 竖版传播图，适合朋友圈、社群、课程或活动 |
| 方法拆解图 | 3:4、4:5 或 16:9 | 内容更完整 | 一张图讲清方法、框架、路径、步骤 |
| 分镜故事图 | 16:9、3:4 或 4:5 | 讲清楚重点 | 2-4 格讲过程、前后变化或踩坑复盘 |
| 其他用途 | 按情况判断 | 按情况判断 | 头像、视频封面、插图、社群贴纸等 |

## 知识卡形态匹配器

| 内容结构 | 优先形态 | 适合尺寸 | 何时使用 |
|---|---|---:|---|
| 强观点、金句、反常识判断 | 观点卡 / 报纸卡 | 3:4、4:5、16:9 | 需要让用户先记住一句话 |
| 路径、阶段、增长闭环 | 路线图卡 / 地图卡 | 16:9、3:4 | 有起点、节点、终点和反馈回流 |
| 方法、框架、模型 | 白板讲解卡 / 方法拆解卡 | 16:9、3:4、4:5 | 用户 IP 适合站旁边讲解 |
| 今天就能做的步骤 | 行动工单卡 / Checklist 卡 | 16:9、3:4 | 知识要转成任务、清单、下一步 |
| 课程、章节、模块 | 课程总览卡 / 章节封面卡 | 16:9、9:16、3:4 | 有课程包装、章节号、学习路径 |
| 评分、诊断、对比评估 | 诊断评分卡 / 雷达卡 / 体检单 | 16:9、3:4 | 有分数、维度、问题、建议 |
| 产品界面、账号主页、截图素材 | 产品截图批注卡 / 界面讲解卡 | 16:9、4:5 | 需要基于真实界面批注和解释 |
| 抽象系统、复杂关系 | 立体沙盘卡 / 系统模型卡 | 16:9 | 需要把概念做成可观察的模型 |
| 前后变化、踩坑复盘 | 分镜知识卡 / 对比卡 | 16:9、3:4、4:5 | 有过程、转折、失败和修正 |

如果内容同时命中多个形态，优先选择最能帮助读者看懂结构的形态，而不是最热闹的形态。

## 图内信息量

对用户说“图内信息量”：

| 用户侧说法 | 内部映射 | 解释 |
|---|---|---|
| 画面更空 | low | 只讲一个点，文字很少，留白强，优先用于手绘插图模式 |
| 讲清楚重点 | medium | 一个主画面 + 几个短标签，适合轻量知识卡或中信息插图 |
| 内容更完整 | high | 放步骤、路径、模块、风险或行动，适合知识卡模式 |
| 完整海报 | extreme | 信息最多，适合课程、活动、方法总览，但仍要保证阅读顺序 |

插图模式只使用前三档，不使用“完整海报”。知识卡模式可以使用后三档，但不要为了填满画面而塞信息。

## 推荐格式

```text
我建议这样做：

视觉模式：知识卡模式
首选：图文号知识卡
- 结构/形态：观点卡 / 路线图卡 / 白板讲解卡 / 或为本内容定制的新形态
- 默认尺寸：3:4
- 图内信息量：讲清楚重点
- 出图效果：一张竖版知识卡，画面中有你的手绘 IP 角色、一个主视觉隐喻、3-5 个短标签/箭头。
- 推荐原因：这段内容是一个清晰观点，适合做成一张可保存的知识卡。
- 角色分工：你的 IP 负责主讲/引导/批注；Agents 或执行者只做辅助演绎。

还可以选：
1. 长文配图：画面更空，文字很少，只用一个动作或场景表达核心观点。
2. 方法拆解图：信息更多，会把步骤、路径、关键点画出来。
```
