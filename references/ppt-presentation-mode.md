# PPT 演讲模式

PPT 演讲模式是 `ip-diagram-creator` 的第三个一级视觉模式。它用于有明确个人 IP 角色的直播分享、课程课件、个人演讲、主题分享和案例复盘。

核心规则：

```text
必须像 PPT 页面，但不能像廉价模板、培训讲义、信息图堆叠或商业海报。
```

## 适用范围

适用于：

- IP 演讲 PPT。
- 直播分享课件。
- 课程讲解页。
- 主题演讲和案例复盘。
- 全图化手绘 PPT 页面。

不适用于：

- 没有个人 IP 角色的普通商业汇报。
- 正式数据报表、客户报价、多人协作编辑型 PPT。
- 非手绘风格的发布会、科技风、商业模板风或杂志风 PPT。

## 工作流

```text
主题 / 大纲 / 逐字稿 / 旧 PPT
-> 判断是否有已确认 IP 角色
-> 理解演讲目标、受众和口播主线
-> 输出整套 PPT 导演规划卡
-> 用户调整页数、节奏、密度、人物出现频率
-> 选 1-2 页代表页做样张
-> 确认字体、人物、密度和风格
-> 分批生成页面
-> 生成 contact sheet
-> 做单页 QA 和整套 QA
-> 返修问题页
```

硬规则：

- 不先批量出图。
- 不在没有导演规划的情况下逐页自由发挥。
- 不把每一页都做成知识卡。
- 不把每一页都做成高密度机制图。

## 导演规划卡

出图前先输出整套导演规划卡：

```text
页码：
页面标题 / 暂定标题：
页面类型：
视觉权重：anchor / dense / breathing
沟通任务：
核心观点：
主视觉：
版式结构：
图文比例：
信息密度：
是否放 IP 人物：
IP 人物角色：
IP 人物位置和比例：
是否放辅助 Agent / 人物：
辅助角色职责：
图内文字策略：
前后页衔接：
QA 风险：
```

导演规划用于提前发现连续高密度页、人物过多、风格断裂和听众节奏问题。

## Page Card

用户确认导演规划后，每页生成前形成 page card：

```text
页码：
页面类型：
视觉权重：
沟通任务：
核心观点：
图内标题：
图内文字：
图文比例：
主视觉：
IP 角色职责：
辅助 Agent / 人物职责：
页面密度：
前后页衔接：
禁止项：
返修重点：
```

page card 是生成 prompt 和出图后 QA 的依据。

## 页面类型库

| 页面类型 | 用途 |
|---|---|
| 封面页 | 定主题、定气质、建立第一印象 |
| 身份 / 信任页 | 说明 IP 角色为什么有资格讲 |
| 目录 / 路线页 | 告诉听众接下来怎么走 |
| 章节转场页 | 切换部分，帮助听众换脑子 |
| 场景页 | 用画面讲真实情境 |
| 痛点页 | 放大一个问题，引发共鸣 |
| 大判断页 | 一句话立观点 |
| 类比页 | 用隐喻解释复杂概念 |
| 机制页 | 解释系统如何运转 |
| 架构页 | 展示模块、层级、能力关系 |
| 手术页 | 把流程 / 案例拆开讲关键节点 |
| 演示页 | 模拟真实操作、输入、输出、确认 |
| 案例页 | 讲具体案例的背景、动作、结果 |
| 数据 / 图表页 | 展示数字、趋势、对比、证据 |
| 时间线 / 路线图页 | 展示阶段、演进、计划 |
| 组织 / 角色页 | 展示角色分工、协作关系 |
| 小结页 | 阶段收束，让听众记住 |
| 金句页 | 强化一句核心记忆点 |
| 结束页 | 回扣主题、给行动建议、感谢 |

页面类型由内容决定，不由固定模板决定。

## 视觉权重和节奏

- `anchor`：封面、章节、关键判断、重要转折、结尾。
- `dense`：机制、架构、数据、图表、流程、证据。
- `breathing`：过渡、故事、金句、小结、低密度场景。

节奏规则：

- 不连续 3 页同一种页面类型。
- 不连续 3 页 `dense`。
- 每页只承担一个沟通任务。
- 机制页可以密，但必须被场景页、转场页或大判断页隔开。
- 演讲型 PPT 允许低信息密度页，因为口播会补信息。
- 图比字多，文字服务口播，不替代口播。
- 每 8-12 页至少有一次小结或明显转场。
- 结束页必须回扣开头，不突然换风格。

## 视觉规范

默认风格：

```text
稳定 IP 角色 + 白底手绘图解 + PPT 页面纪律 + 整页图
```

字体规则：

- 主标题稳定、粗黑、商务、清晰。
- 正文、小标签、模块标题尽量统一。
- 手写字只作为少量批注，不承担主信息。
- 如果图里中文字太多，宁可拆页，不硬塞。

颜色规则：

- 默认白底或近白底。
- 主体黑灰线稿。
- 橙色只做重点、路径和批注。
- 蓝色只做结构线、连接线和系统关系。
- 不使用大面积渐变、炫彩或复杂商业模板色。

## IP 角色和辅助角色

- IP 角色必须基于已确认角色资产。
- 角色出现时必须有动作职责，不是站旁边装饰。
- 不要每页都放大人物。
- 人物比例要跟页面类型匹配：封面 / 身份页可大，机制页 / 场景页可小。
- 封面、身份页、关键转折页、结束页可以出现 IP。
- 机制页中 IP 通常在角落，不抢信息。
- 数据 / 图表页一般不出现，除非要作为讲解者。
- 辅助角色用于表达执行、反馈、错误示范、搬运、确认、协作等动作。

## QA

单页 QA：

- 是否符合 page card。
- 是否只有一个核心任务。
- 主标题字体是否稳定、商务、清晰。
- 图内文字是否过多、过小、错字或伪字。
- IP 角色是否像角色锚点。
- IP 角色是否有职责，不是装饰。
- 画面是否像 PPT 页面，而不是知识卡、信息图海报或培训讲义。
- 橙色是否过量。
- 图文比例是否符合规划。

整套 QA：

- 是否像同一个设计系统。
- 字体系统是否乱。
- 人物年龄、气质、服装、发型是否漂移。
- 是否连续多页高密度。
- 页面类型是否有变化。
- IP 是否出现过多或过大。
- 是否有足够转场、场景和 breathing 页。
- 结尾是否回扣开头。
- 是否有某几页像另一个 PPT 混进来。
