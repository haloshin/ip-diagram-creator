# 视觉语言

## 审美底线

克制不等于粗糙，手绘不等于丑。角色必须先是一个让用户愿意长期使用的好看个人 IP，再进入图解场景。

默认审美方向：

- 干净、好看、有记忆点。
- 成人手绘角色，不幼态。
- 线条清爽，有轻微手绘感，但不能潦草。
- 可以有轻微亲和力，但不能像儿童卡通或表情包。
- 可以精致，但不能商业海报、写真肖像或厚涂插画。
- 识别锚点要被美化成稳定符号，而不是机械还原。

如果“图解感”和“好看”冲突，优先保持好看的成人 IP 角色，再减少细节让它适合图解。

## 用户侧叫法

对用户说：

- 极简手绘图解角色
- 白底手绘图解
- 动作隐喻
- 画面更空
- 图内信息量
- Agents
- 执行 Agent
- AI 劳动力
- 执行者
- 反馈来源

不要对普通用户说内部代称，也不要暴露历史项目语境。

## 风格 DNA

- 纯白背景。
- 黑色手绘线稿。
- 轻微抖动的笔触。
- 大量留白。
- 少量红、橙、蓝中文手写批注。
- 干净、怪诞、产品草图感。
- 一张图只讲一个核心动作、结构、状态或隐喻。
- 角色必须参与核心动作。

## 颜色规则

- 黑色：主体线稿、角色、框线、结构、主要文字。
- 橙色：主流程、路径、箭头、从 A 到 B 的移动关系。
- 红色：重点、问题、提醒、结果。
- 蓝色：补充说明、反馈、系统状态、第二层解释。

颜色要克制。蓝色不是每张都必须用。

## 绝对不要

- 不要商业插画。
- 不要 PPT 信息图。
- 不要正式流程图。
- 不要课程课件感。
- 不要可爱卡通海报。
- 不要儿童插画。
- 不要复杂架构图。
- 不要精致扁平插画。
- 不要科技感 UI。
- 不要真实界面截图仿制。
- 不要复杂背景、渐变、阴影、纹理。
- 不要把每个节点都解释清楚。

## 角色规则

用户角色不是吉祥物，不是贴纸，不是站在角落的装饰。它要像一个认真参与系统运转的图解工作者。

判断标准：如果去掉角色，图的核心隐喻还能完全成立，说明角色太装饰，要重写 prompt，让角色成为动作主体。

同时，角色不能因为追求工具感而变丑、变呆、变商务。它应当像一个可持续出现在用户内容里的漂亮手绘 IP：黑白线描为主，少量个人色彩点缀，识别点清晰，表情克制但有生命力。

## 小比例场景角色

手绘插图模式里，用户 IP 经常会变成大场景中的小比例行动角色，例如在船上、港口、工位、机器旁、路径上、流程线旁。此时角色可能只占画面的 10%-20%，但仍然必须像同一个人。

小比例角色最低要求：

- 发型轮廓仍然可识别。
- 眼镜或核心配饰仍然可见。
- 稳定服装不能换色或换风格。
- 体态保持成人创作者比例，不幼态。
- 角色正在做核心动作，比如指路、搬运、操作、批注、拉线、记录。

如果模型在小比例场景里丢掉识别点，返修时要明确要求放大眼镜/配饰、强化发型轮廓、恢复稳定服装，并参考动作 / 表情 / 小比例场景扩展图。

## 个人 IP 边界

用户照片会被转译成“极简手绘图解角色”，不是每张图写实还原本人。

固定的是：

- 发型。
- 眼镜或配饰。
- 稳定服装。
- 体态和动作习惯。
- 表情气质。
- 职业道具。
- 个人色彩点缀。

不要为了追求相似度牺牲图解表达。

## 辅助角色和 Agents

画面里可以出现辅助极简角色，用来表示用户、执行者、失败案例、反馈来源、风险提示或执行 Agent。

规则：

- 用户 IP 是主讲、引导、拆解或核心行动者。
- Agents / 执行 Agent 是辅助执行和反馈来源。
- 如果辅助角色代表 AI 或自动化劳动力，对外写 Agents、执行 Agent 或 AI 劳动力。

知识卡模式里，辅助 Agents 是默认图解语法，不是临时装饰。凡是方法、步骤、流程、对比、案例、风险、行动工单类内容，默认安排 2-6 个 Agents 承担执行、反馈、失败示范、搬运卡片、标记风险、递交结果等动作。只有纯观点海报、极简封面或用户明确要求不要时，才减少或移除。

## 不同风格下的 Agents

如果用户指定不同视觉风格，辅助 Agents 要跟随主风格变化，但不能抢用户 IP 的主角位置：

- **白底手绘**：黑色极简执行 Agent，线条简单，适合搬卡片、举牌、反馈、打叉。
- **像素风**：小像素执行者，保留块状轮廓，用少量像素动作表达执行、传递、测试、报错。
- **Q 版卡通**：小助手形态，亲和但不能过萌，比例小于主角，不要变成幼态主角团。
- **成熟轻写实**：极简符号人或工作流小人，偏任务图标和流程角色，不要做得幼稚或抢戏。

无论哪种风格，用户 IP 负责主讲、拆解、批注、调度；Agents 负责执行、反馈、示错、搬运和风险提示。
