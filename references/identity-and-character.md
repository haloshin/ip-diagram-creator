# 身份提取与角色固定资产

## 核心字段

无论用户用什么方式开始，都围绕这些字段提取：

| 字段 | 说明 | 来源 |
|---|---|---|
| 外形参考 | 本人照片、头像、主页截图、视频封面 | 用户授权上传图片或截图 |
| 称呼 | 用户希望别人怎么叫他 | 主页昵称、简介、用户文字 |
| 领域 / 职业 | 用户做什么、主要内容领域 | 简介、标签、标题、课程介绍 |
| 简介 | 我是谁、帮谁、解决什么 | 用户段落、主页简介、账号说明 |
| 倾向气质 | 角色给人的感觉 | 用户选择，或根据材料推断 |
| 识别锚点 | 发型、眼镜、服装、体态、道具、色彩 | 图片和文字材料 |
| 禁区 | 不想要的风格、服装、表情、年龄感 | 用户说明或合理推断 |

## 输入不要求表单

用户可能只发：

- 一张照片。
- 社媒主页截图。
- 一段自我介绍。
- 课程页截图。
- 账号简介。
- “我做 XXX，想做个人 IP 图解角色。”

先提取，不要要求重填表。

## 已知 / 推断 / 缺失

输出时区分三类：

```text
已明确：
- 称呼：
- 领域：
- 简介：

我先推断：
- 气质：
- 识别锚点：
- 适合道具：

还缺：
- [只列真正阻塞的字段]
```

只有这些情况需要追问：

- 照片看不清。
- 没有任何外形参考。
- 简介太空，无法判断领域。
- 气质明显冲突。
- 用户要求强相似度或指定穿搭。
- 用户明确有不想要的元素。

每次只问一个最关键问题。

## 三张角色固定资产

角色建档要生成三张可长期复用的固定资产：

1. `01_角色主锚图`
2. `02_角色规范说明图`
3. `03_动作_表情_小比例场景扩展图`

三张图分工不同：主锚图负责“像谁”，规范说明图负责“别跑偏”，动作扩展图负责“会动、会变小、会进入场景”。

## 01 角色主锚图

用途：后续普通内容图默认传这一张。

版式要求：

- 人物要大，不能被大量说明文字挤小。
- 至少包含一个大主形象，优先 3/4 视角或全身。
- 补充 2-3 个清晰视角：正面、3/4、侧面。
- 只放少量身份锚点标签。

必须看清：

- 发型。
- 眼镜或核心配饰。
- 稳定服装。
- 体态比例。
- 常态表情和气质。
- 1-2 个个人色彩点缀。

## 02 角色规范说明图

用途：给用户确认，也给 Agent 做审核和返修。它不是每次生图的默认主参考。

版式要求：

- 保留一个足够大的主形象，避免全是小脸。
- 分区说明：必须保留、禁止偏移、常用道具、配色、使用建议。
- 禁止偏移必须画出来并打叉，例如错误服装、错误眼镜、幼态化、商务化、写实化。
- 表情参考可以有，但不能多到影响身份一致性。

## 03 动作 / 表情 / 小比例场景扩展图

用途：用于复杂动作、多姿态、多表情，以及手绘插图模式里的小比例场景人物。

如果用户角色属于明确课程、行业或长期内容系列，动作/小比例场景必须优先来自该领域的高频真实场景，而不是照搬通用动作清单。

小比例场景角色最低要求：

- 发型轮廓仍然可识别。
- 眼镜或核心配饰仍然可见。
- 稳定服装不能换色或换风格。
- 体态保持成人创作者比例，不幼态。
- 角色正在做核心动作，而不是站旁边装饰。

## 角色确认问题

三张资产生成后，让用户确认：

```text
请确认这三张能否作为你的长期图解角色资产：

1. 主锚图：像不像你希望长期被记住的样子？
2. 规范说明图：哪些必须保留、哪些禁止偏移是否准确？
3. 动作 / 表情 / 小比例场景扩展图：这些动作和小比例角色能否用于后续文章配图和知识卡？
```

用户确认前，不进入内容图解。

## 角色档案字段

确认后，形成角色档案：

```text
display_name:
field:
bio:
personality:
identity_anchors:
visual_rules:
main_anchor_asset:
spec_asset:
action_expression_small_scene_asset:
action_rules:
small_scene_rules:
expression_rules:
props:
color_accents:
avoid:
reference_priority:
```

用户不需要理解字段格式。对用户展示为自然语言摘要即可。
