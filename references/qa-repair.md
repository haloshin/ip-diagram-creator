# QA 和返修

## QA 检查顺序

1. **角色稳定**
   - 是否像角色主锚图。
   - 是否保留发型、眼镜/配饰、服装、体态、气质。
   - 是否像同一个长期 IP 角色。
   - 如果不像角色主锚图，直接不合格，先修角色。

2. **动作核心**
   - 角色是否在做核心动作。
   - 如果去掉角色，图是否还能完全成立；如果能，说明角色太装饰。

3. **内容准确**
   - 是否表达了用户确认的核心观点。
   - 是否遗漏必须保留的词。

4. **模式匹配**
   - 如果选的是手绘插图模式，是否仍然有明显留白和单一主隐喻。
   - 如果选的是知识卡模式，是否有清楚标题、阅读顺序和信息结构。
   - 知识卡形态是否真的服务内容，而不是硬套参考图。
   - 用户 IP 和 Agents / 执行者的分工是否清楚：主讲/引导 vs 穿插/执行/案例。
   - 用户可见文字是否误写内部代称；如出现，必须改掉。

5. **文字可读**
   - 图内文字是否清楚。
   - 是否有严重错字。
   - 是否文字过多。

6. **视觉风格**
   - 是否白底。
   - 是否手绘。
   - 是否留白足够。
   - 红橙蓝是否克制。
   - 是否足够好看，用户是否愿意长期使用这个 IP。

7. **失败信号**
   - 太像 PPT。
   - 太商业。
   - 太幼态可爱。
   - 太精致插画。
   - 太丑、太粗糙、太呆。
   - 太商务头像感。
   - 把插图做成了满屏知识卡。
   - 把知识卡做成了只有一个隐喻的普通配图。
   - 硬套参考图，没有根据当前内容设计结构。
   - 背景太复杂。
   - 复刻旧图构图。
   - 手绘插图里的小比例用户 IP 丢掉发型、眼镜/配饰或稳定服装。

## 用户反馈映射

| 用户反馈 | 返修方向 |
|---|---|
| 不像我 | 强化发型、眼镜、服装、脸部轮廓和体态 |
| 识别特征不明显 | 放大稳定锚点，减少无关装饰 |
| 太可爱 / 太幼态 | 去掉萌化比例和夸张表情，改成人创作者感 |
| 太严肃 / 太商务 | 增加松弛姿态、轻微幽默、手绘草图感 |
| 太普通，没有记忆点 | 增加职业道具和更具体的动作隐喻 |
| 太丑 / 太粗糙 | 提升线条质量、比例和角色美感，保留手绘但不要潦草 |
| 太商务头像感 | 降低正式头像和顾问感，增加手绘 IP 的松弛感和亲和力 |
| 太复杂，不像手绘草图 | 减少模块、箭头、标签和背景元素 |
| 太精致 | 降低商业插画感，回到白底线稿和轻微抖动笔触 |
| 道具不对 | 替换为领域相关道具 |
| 气质不对 | 调整表情、姿态、动作和色彩点缀 |
| 颜色不喜欢 | 保持黑白底，只替换 1-2 个点缀色 |
| 小比例人物不像我 | 参考动作 / 表情 / 小比例场景扩展图，放大眼镜/配饰，强化发型轮廓，恢复稳定服装 |
| 图内信息太少 | 升一级图内信息量，增加标签或模块 |
| 图内信息太多 | 降一级图内信息量，保留一个核心动作 |
| 这不是知识卡，只像配图 | 增加标题、模块、阅读顺序、关键结论和适合内容的卡片形态 |
| 这不像插图，太满了 | 回到手绘插图模式，减少模块和文字，只保留一个主隐喻 |
| 太像套模板 | 重新根据内容结构选择或创造卡片形态，保留风格但换构图 |
| 文字不准 / 看不清 | 局部重写文字，减少字数，保留必须词 |
| 图里写了内部代称 | 删除该词，按语义改成 Agents、执行 Agent、用户、执行者、反馈来源或风险提示 |

## 返修输出格式

```text
本轮保留：
- [角色锚点]
- [参考角色资产]
- [核心观点]
- [图类型和尺寸]

本轮调整：
- [具体改动]

返修 prompt：
[可直接复制或用于图像生成工具的 prompt]
```

不要每次返修都重做角色。只有用户明确说“角色方向不对 / 重来”时，才回到角色固定资产阶段。
