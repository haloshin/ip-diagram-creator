# 内容理解、shot list 与一句话扩展

## 总原则

先判断输入源，再决定拆法。不要先让用户选场景、尺寸、图内信息量。

内容输入后，先判断它更适合：

- **手绘插图模式**：适合一个认知锚点、一个动作隐喻、文章中间配图、观点留白表达。
- **知识卡模式**：适合独立传播、收藏、教学、拆解、课程、诊断、步骤、路径、截图批注、行动清单。
- **PPT 演讲模式**：适合有个人 IP 角色的直播分享、课程课件、主题演讲、案例复盘和全图化手绘 PPT 页面。

同一篇长文可以同时包含多种模式：有些段落适合插图，有些段落适合知识卡；如果用户要做演讲或课件，则先进入 PPT 演讲模式，做整套页面规划。

## 输入源类型

### 长文 / 文章 / 文档 / 链接 / 截图

- 先消化正文。
- 提炼核心观点。
- 找认知转折。
- 判断哪些段落适合用图解释。
- 判断哪些内容只适合文字，不需要图。
- 输出 shot list，并为每张图判断视觉模式和推荐形态。

默认推荐 4-8 张。文章很短时 1-3 张；长文也不要轻易超过 9 张。

### 演讲主题 / 课程大纲 / 逐字稿 / 旧 PPT

当用户明确要做 PPT、课件、直播分享、主题演讲、整套页面或逐页出图时：

- 判断为 PPT 演讲模式。
- 先理解演讲目标、受众和口播主线。
- 不直接给普通 shot list。
- 先输出整套导演规划卡。
- 建议 1-2 页样张，用于确认字体、人物和信息密度。
- 用户确认样张后再分批生成页面。

### 短观点 / 笔记 / 一段话

通常推荐 1-3 张：

- 一个主观点：如果用于正文中，推荐手绘插图；如果用于单独发布，推荐观点卡/图文号知识卡。
- 多个并列观点：可以做卡组、方法拆解卡、行动工单卡或自定义知识卡形态。
- 情绪/状态表达：可做分镜故事图。

### 一句话主题

不要直接出图。先扩展成图解脚本，再让用户确认。

常见类型：

| 类型 | 扩展公式 | 适合形式 |
|---|---|---|
| 观点句 | 误区 -> 转折 -> 正确做法 | 图文号知识卡、长文配图、前后对比 |
| 方法句 | 起点 -> 步骤 -> 结果 | 方法拆解图、路径图、卡组 |
| 痛点句 | 卡住 -> 原因 -> 解法 | 分镜故事图、前后对比图、知识卡 |
| 概念句 | 定义 -> 类比 -> 关键组成 | 知识卡、方法拆解图、长文配图 |
| 宣传句 | 对象 -> 价值 -> 行动 | 手机海报、图文号知识卡 |

必须区分：

- 用户原话。
- 我补充的推断。
- 可修改项。

如果推断较多，要提醒用户先确认，不要直接出图。

## 内容判断输出

每次内容输入后，先输出：

```text
我先判断这段内容适合这样处理：
- 输入源类型：
- 适合图解的核心：
- 推荐视觉模式：
- 推荐知识卡形态（如适用）：
- 推荐图数：
- 首选图类型：
- 备选方向：
```

## Shot list 规则

长文处理流程：

```text
读正文 / 链接 / Markdown / 截图
-> 提炼核心观点
-> 找认知转折
-> 判断适合图解的段落
-> 输出 shot list
```

不要平均配图。优先选择认知锚点：

- 核心判断。
- 两个断点。
- 输入输出闭环。
- 分流。
- 前后对比。
- 承接路径。
- 常见坑。
- 角色状态变化。

每张图写清楚：

```text
图序号：
建议放置位置：
图的主题：
核心意思：
视觉模式：
结构类型：
知识卡形态（如适用）：
推荐图类型：
推荐尺寸：
图内信息量：
角色动作：
角色资产参考：
用户 IP / Agents 分工：
建议元素：
建议中文标注：
为什么这张值得画：
```

## 结构类型

一次选择一种结构即可，不要混太多：

- Workflow 流程：输入 -> 处理 -> 输出。
- 系统局部：3-5 个核心模块。
- 前后对比：混乱/有序，手动/自动，分散/收拢。
- 角色状态：2-4 个小状态。
- 概念隐喻：一个大的怪物件或机器。
- 方法分层：一层层盒子，但不要正式金字塔。
- 地图路线：一条弯曲路径，少量节点。
- 分镜故事：2-4 个小场景。

## 知识卡形态判断

知识卡不是固定模板库。参考形态包括但不限于：

- 观点卡 / 报纸卡：强观点、反常识、金句。
- 路线图卡 / 地图卡：路径、阶段、增长闭环。
- 白板讲解卡：方法、框架、三段式解释。
- 行动工单卡：任务清单、今天能做什么。
- 课程总览卡 / 章节封面卡：课程、模块、章节。
- 诊断评分卡：评分、维度、问题和建议。
- 产品截图批注卡：真实界面、账号主页、产品截图。
- 立体沙盘卡 / 系统模型卡：抽象系统、复杂关系。
- 分镜知识卡：过程、踩坑、前后变化。

如果已有形态不合适，就为当前内容创造新形态，并在推荐原因里解释它如何帮助读者看懂结构。

## 原创隐喻

每次都从当前内容重新发明隐喻：

1. 把抽象概念换成物理动作：卡住、漏掉、变重、分拣、沉淀、发酵、开门、折叠、拆包、回流。
2. 把系统结构换成低科技物件：坏机器、纸箱、抽屉、水管、邮筒、怪表盘、秤、井、梯子、怪工位。
3. 让用户 IP 角色承担动作：拉线、搬运、守门、修补、称重、记录、把东西塞进怪装置。

不要复刻旧案例构图。
