# 示例：知识卡模式判断

## 用户输入

```text
我已经有个人 IP 角色了。现在有一段内容：我们做了一个 AI 产品，但增长卡住了，我想解释给用户：不是功能不够，而是缺少从被看见、被理解、被信任到被使用的路径。准备用在图文号，也可能做课程里的一页。你判断该做什么形式，不要只套我给你的参考图。
```

## 期望处理

Agent 应判断为知识卡模式，而不是普通插图。

推荐时应包含：

- 首选知识卡形态，例如路线图卡、白板讲解卡、行动工单卡，或为该内容创造的新形态。
- 推荐依据：内容包含路径、阶段、用户理解过程和转化承接。
- 尺寸建议。
- 图内信息量。
- 用户 IP 主讲 / 引导 / 批注方式。
- Agents / 执行者如何辅助演绎。
- 首选和备选方案。

不要复刻参考图构图，也不要把知识卡简单解释成“高信息量插图”。
