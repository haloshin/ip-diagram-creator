# 示例：长文 shot list

## 用户输入

```text
我已经确认了我的手绘 IP 角色。现在我给你一篇 3000 字文章，主题是 AI 产品做完以后为什么还需要内容和渠道。请分析这篇文章适合配几张图，先给 shot list，不要直接全生成。
```

## 期望处理

Agent 应先消化文章，不直接出图。推荐 4-8 张 shot list，每张包含：

- 图序号。
- 建议放置位置。
- 图的主题。
- 核心意思。
- 视觉模式。
- 结构类型。
- 知识卡形态（如适用）。
- 推荐图类型。
- 推荐尺寸。
- 图内信息量。
- 角色动作。
- 角色资产参考。
- 用户 IP / Agents 分工。
- 建议元素。
- 建议中文标注。
- 为什么这张值得画。

同一篇文章可以同时包含手绘插图模式和知识卡模式，不要把所有图都做成同一种样子。
